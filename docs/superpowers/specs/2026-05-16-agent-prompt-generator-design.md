# Agent Prompt Generator — Design Spec
**Task:** TASK-013
**Date:** 2026-05-16
**Status:** Approved

---

## Overview

A new service inside `app/prompt-engine/` that uses Claude Sonnet (structured output via tool use) to generate `AGENT_PROMPT_COUNT` (default 25) targeted prompts per client, based on that client's brand profile fields added in TASK-012. Generated prompts are saved as tenant custom prompts scoped to the specific client.

---

## Schema Changes

Two additive columns added to the `Prompt` model in `prisma/schema/schema.prisma`. No existing rows are affected (both have safe defaults).

```prisma
source    String  @default("platform")  // platform | agent | custom
client_id String?                        // null for platform/tenant prompts; set for agent prompts

client    Client? @relation(fields: [client_id], references: [id])
@@index([client_id])
```

Migration name: `add_prompt_source_and_client_id` (additive, no breaking changes).

Values for `source`:
- `platform` — seeded by super_admin (existing rows, default)
- `agent` — generated by `AgentPromptGeneratorService`
- `custom` — manually created by tenant_admin (future use)

## New File

`app/prompt-engine/agent-prompt-generator.service.ts`

### Input
- `clientId: string`
- `user: RequestUser` (for tenant-scoped access control)

### Flow

1. Fetch client from DB — tenant-scoped (same guard as `triggerClientRun`). Throw `NotFoundException` if not found.
2. **Idempotency check:** count existing prompts where `source='agent'` AND `client_id=clientId`. If count >= `AGENT_PROMPT_COUNT` → return `{ skipped: true, reason: 'prompts already generated' }`.
3. Validate `brand_description` is present. Throw `BadRequestException` if missing.
4. Build system + user message for Claude using:
   - `client.brand_description`
   - `client.brand_keywords` (JSON array)
   - `client.digital_handles` (JSON object)
   - vertical name (fetched via `client.vertical_id`)
5. Call Anthropic SDK `messages.create` with a `store_prompts` tool schema:
   ```ts
   tools: [{
     name: 'store_prompts',
     input_schema: {
       type: 'object',
       properties: {
         prompts: {
           type: 'array',
           items: {
             type: 'object',
             properties: {
               text: { type: 'string' },
               category: { type: 'string' },
               intent_type: { type: 'string' },
               buyer_stage: { type: 'string' },
               priority: { type: 'number' },
             },
             required: ['text', 'category', 'intent_type', 'buyer_stage', 'priority'],
           },
         },
       },
       required: ['prompts'],
     },
   }]
   ```
6. Extract `tool_use` block from response, parse `input.prompts`.
7. Bulk insert via `prisma.prompt.createMany()`:
   - `is_custom: true`
   - `source: 'agent'`
   - `client_id: clientId`
   - `tenant_id: client.tenant_id`
   - `vertical_id: client.vertical_id`
   - `is_active: true`
8. Return `{ generated: N, promptIds: [...] }`.

### Config

| Env var | Default | Purpose |
|---|---|---|
| `AGENT_PROMPT_COUNT` | `25` | Number of prompts to generate per client |
| `ANTHROPIC_API_KEY` | required | Anthropic SDK auth (already in use by claude.adapter.ts) |

Model: `claude-sonnet-4-6` (current session model, matches stack).

## Module Changes

`prompt-engine.module.ts` — add `AgentPromptGeneratorService` to `providers[]`.

---

## Controller Change

`prompt-engine.controller.ts` — add one endpoint:

```ts
@Post('clients/:clientId/generate-prompts')
@Roles(UserRole.super_admin, UserRole.tenant_admin)
generatePrompts(@Param('clientId') clientId: string, @Req() req: Request) {
  return this.agentPromptGenerator.generateForClient(clientId, req.user as RequestUser);
}
```

---

## Error Handling

| Condition | Response |
|---|---|
| Client not found / wrong tenant | `404 NotFoundException` |
| `brand_description` is null/empty | `400 BadRequestException` |
| Claude returns fewer than 25 prompts | Insert what was returned, log warning |
| Claude API fails | Bubble up as 500 |
| Already have 25+ agent prompts | `{ skipped: true }` (200, not an error) |
